前端代码规范
发表于 2019-12-29| 更新于 2019-12-30
字数总计: 4.7k|阅读时长: 21 分钟| 阅读量:

良好的代码书写规范是一个习惯，能提高代码的可读性和一个团队的开发效率。

该文采用腾讯alloyteam团队的前端代码规范，该文章有点偏长，但花费时间并不长，学习起来也不困难。
若只是用于查看或参考某部分的，可以通过左侧目录导航跳转到相应的模块（手机端用户请点击右侧悬浮的目录按钮）

命名规范

项目命名

项目起名全部采用小写方式，使用下划线做分割

例如： my_project_name

目录命名

同项目命名，如果有复数结构（多个），则在后面加上s
例如：Catalogs, my_Catalogs

JS文件命名

同项目命名
例如： my_utils.js

CSS和编译成css的scss文件等命名

同项目命名
例如： my_css.css my_scss.scss

HTML文件命名

同项目命名
例如： my_html.html

HTML篇

语法

  • 缩进使用soft tab(4个空格)

  • 嵌套节点使用缩进（子节点缩进）

  • 属性（行内属性）使用双引号，不要用单引号（单引号可能会引发错误）

  • 属性名称使用小写，用中划线做分割（例如：name=”name-name”）

  • 不要在自动闭合标签结尾处使用斜线（HTML5规范表示是可选的）

  • 不要忽略可选的关闭标签（例如：

    • 和）

    自动闭合标签：

  • meta: 用于设置页面元信息

  • base 设置网页所有连接的相对目录

  • br 换行

  • hr 水平线

  • img 图像

  • input 表单元素

  • col 在表格table中定义一个或多个列的属性

  • iframe 定义框架的一个窗口（已遗弃）

  • link 定义文档与外部资源的关系的链接

  • area 标签定义图像映射内部的区域（图像映射指的是带有可点击区域的图像）。

  • param 元素允许您为插入 XHTML 文档的对象规定 run-time 设置，也就是说，此标签可为包含它的

  • object 或者 applet 标签提供参数。

  • embed HTML5 中新增的,标签定义了一个容器，用来嵌入外部应用或者互动程序（插件）。

  • source 标签为媒体元素（比如 和 ）定义媒体资源。

HTML5 doctype

页面开头doctype启用标准模式，使其能在每个浏览器中能一致的展现
doctype可不区分大小写，但是应该大写

lang属性

html标签上加上lang属性，会告诉语音工具和翻译工具怎么发音和翻译。
lang的语言代码可点击更多查看

字符编码

用meta标签声明一个字符编码，让浏览器更快速的确定该网页内容的渲染方式，通常指定为’UTF-8’

IE兼容模式

用meta标签声明用哪个版本的浏览器渲染页面，不同的浏览器下会触发不同的渲染模式

引入css、js文件

引入css、js文件时不需要写明type，HTML5规范会默认为text/css和text/javascript

行内标签属性顺序

标签内属性应该按照特定顺序（同级别可不分先后）

  1. class
  2. id
  3. name
  4. data-*
  5. src , for , type , href , value , max-length , max , min , pattern
  6. placeholder , title , alt
  7. aria-* , role
  8. required , readonly , disabled

boolean属性

boolean属性可以不需要声明值
boolean属性存在则取值为true，不存在取值为false

减少标签数量

尽量减少多余的父节点和嵌套关系

CSS篇

缩进

使用soft tab(4个空格)

分号

每个属性末尾都需要加分号

空格

需要空格的情况如下

  • { 前
  • 属性值前
    （rgb(0, 0, 0)）
  • 选择器前后
    .css > .next
  • !important的!前
    color: red !important
  • @else 前后
  • 注释 / 前后
    /* 注释 */
  • 多个属性
  • 多个分隔符,后
    rgb(0, 0, 0)

不需要空格的情况如下

  • 属性名后
  • !important 的! 后
  • 属性值中()前后
  • 行末不需要空格
  • 多个分隔符,前

注释

缩进与下行代码保持一致
与代码同行时，则与代码保持一个空格

引号

最外层使用双引号
属性值要用引号

命名

  • 类名使用小写，中划线分割
  • id使用驼峰式
  • scss中变量、函数、placeholder使用驼峰式

属性声明顺序

相关属性成组，各组之间空行分隔

下面是属性的顺序，由于太长，我简单概况下比较容易理解

  1. 确定这个盒子的特性（float、display等）
  2. 确定这个盒子的位置（position）
  3. 确定这个盒子的外形
  4. 确定这个盒子里面放的内容
  5. 确定这个盒子里面内容张什么样

颜色

16进制用小写字母，尽量用简写 (xxyyzz形式的可简写成xyz)

属性简写

属性有拆分，可简写

媒体查询

媒体查询规则尽量和他们相关的规则靠近，不要把他们放进独立的文件或者文档的底部

杂类

不允许有空的规则；

元素选择器用小写字母；

去掉小数点前面的0；

去掉数字中不必要的小数点和末尾的0；

属性值’0’后面不要加单位；

同个属性不同前缀的写法需要在垂直方向保持对齐，具体参照右边的写法；

无前缀的标准属性应该写在有前缀的属性后面；

不要在同个规则里出现重复的属性，如果重复的属性是连续的则没关系；

不要在一个文件里出现两个相同的规则；

用 border: 0; 代替 border: none; ；

选择器不要超过4层（在scss中如果超过4层应该考虑用嵌套的方式来写）；

发布的代码中不要有 @import ；

尽量少用’*’选择器。

JavaScript篇

缩进

使用soft tab(4个空格)

单行长度

单行长度不能超过80，如果编辑器开启word wrap可以不用考虑单行长度

分号

  1. 声明变量
  2. 表达式
  3. return
  4. throw
  5. break
  6. continue
  7. do-while

空格

以下情况不需要空格：

  • 对象的属性名后
  • 前缀一元运算符后
  • 后缀一元运算符前
  • 函数调用括号前
  • 函数声明或函数表达式，括号前面不需要
  • 数据[]括号前
  • 对象{}括号前
  • 运算符()括号前

以下情况需要空格

  • 二元运算符前后
  • 三元运算符’?:’前后
  • 代码块’{‘前 下列关键字前： else , while , catch , finally
  • 下列关键字后： if , else , for , while , do , switch , case , try , catch , finally , with , return , typeof
  • 单行注释’//‘后（若单行注释和代码同行，则’//‘前也需要），多行注释’*’后
  • 对象的属性值前
  • for循环，分号后留有一个空格，前置条件如果有多个，逗号后留一个空格
  • 无论是函数声明还是函数表达式，’{‘前一定要有空格
  • 函数的参数之

空行

以下情况需要空行

  • 变量声明后（若在代码块的最后一行，可不用）
  • 注释前（若在代码块的最后一行，可不用）
  • 代码块后
  • 文件最后

换行

行末必须有,或者运算符
以下情况无需换行

  • else、catch、finally关键字后
  • 代码块 { 前

以下情况需换行

  • 代码块 { 后和 } 前
  • 变量赋值后

单行注释

双斜线后需要跟一个空格
缩进与下行代码保持一致
可放于代码行末，与代码间隔一个空格

多行注释

以下情况使用

  • 难以理解的代码块
  • 需要解释错误的代码块
  • 特殊代码块

文档注释

含有变量、函数、类等的注释

引号

最外层统一使用单引号

变量命名

  • 标准变量采用驼峰式（除对象属性外）
  • ‘ID’,’URL’在变量名中全大写
  • ‘Android’在变量名中首字母大写
  • ‘iOS’在变量名中首字母小写，其余大写
  • 常量全大写，下划线链接
  • 构造函数是首字母大写
  • jQuery对象必须以’$’开头命名

变量声明

一个函数作用域中的所有变量声明全放在函数头部，不用重复使用var声明

函数

  • 函数声明或是函数表达式，’(‘前不空格,’{‘前必须空格
  • 函数调用括号前不需要空格
  • 立即执行函数外层包一层括号
  • 参数之间用,分割，且在逗号后有一个空格
  • 不要给给inline function（内联函数）命名

数组、对象

对象属性名不需要加引号
对象内容缩进
数组、对象最后不要有都好

括号

在以下关键字内必须要大括号，即使是一行代码：if， else， for， while， do， switch， try， catch， finally， width

null使用

null的使用场景

  • 初始化一个即将可能被赋值为对象的变量
  • 与已经初始化的变量做比较
  • 作为一个参数为对象的函数调用
  • 返回对象的函数返回值
    以下不适用
  • 不能用来做判断函数调用有无传参
  • 不能作为和未初始化的变量做比较

undefined

不能直接使用undefined作变量判断
使用typeof 与 字符串’undefined’做判断

jsHint

  • 用’===’,’!==’代替’==’,’!==’
  • for-in一定要进行hasOwnProperty的判断
  • 不要在内置对象原型上添加方法
  • 不要在内层作用域代码声明变量后却使用外层的同名变量
  • 变量必须先声明再使用
  • 不要在单单使用构造函数，记得将其赋值给一个变量
  • 不要在同一作用域声明同名变量
  • 不要在一些不需要的地方加括号
  • 不要使用未声明变量
  • 没有使用到的变量不要声明
  • 不要在作比较的地方做赋值
  • debugger不能出现在提交的代码里
  • 数组不要存在空元素
  • 不要在循环中声明变量
  • new和构造函数分开

杂类

  • 不要混用空格和tab
  • 不要在一处使用多个tab和space
  • 对上下文的this的引用，只能用’that’,’_this’,’self’来命名
  • 行尾不要有空白字符
  • switch要对没有default特殊说明
  • 不允许有空代码快
